# Security Policy

## Supported Versions

Only the latest stable release builds are supported with security patches.

## Reporting a Vulnerability

If you identify a security vulnerability in sqlfluff (such as execution sandbox escapes in sql templating, path transversal flaws during parsing, or dependencies exploits), please do **not** open a public issue.

Please report vulnerabilities privately by emailing the maintainers or using GitHub's private vulnerability reporting feature.
